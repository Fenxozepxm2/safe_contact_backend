## О проекте

«Безопасный контакт» — это информационная система, которая автоматизирует взаимодействие между клиентами и психологами. Платформа позволяет клиентам находить специалистов, записываться на консультации, оплачивать их и проводить видеовстречи — всё в едином безопасном пространстве. Бэкенд построен на микрофреймворке Flask, использует PostgreSQL в качестве основной СУБД и интегрируется с внешними сервисами:

*   **Logto** — аутентификация и управление доступом (OIDC, JWT).
*   **Cal.com** — синхронизация расписания специалистов.
*   **ЮKassa** — приём и обработка платежей (эквайринг, вебхуки).
*   **Jitsi Meet** — организация видеоконференций.

## Архитектура приложения

Проект построен по модульному принципу: вся функциональность разделена на независимые Blueprints. Это упрощает поддержку и масштабирование.

## Основные эндпоинты

| Метод | Эндпоинт                     | Описание                                                        |
| ----- | ---------------------------- | --------------------------------------------------------------- |
| GET   | `/auth/login`                | Вход в систему (перенаправление на logto)                       |
| POST  | `/auth/change-role`          | Сменить роль пользователя (также отвечает за создание профилей) |
| POST  | `/slots/create`              | Создать слот(ы) (специалист)                                    |
| GET   | `/slots/get`                 | Получить все слоты текущего специалиста                         |
| POST  | `/clients/make-appointment/` | Забронировать слот (клиент)                                     |
| PUT   | `/specialists/update`        | Обновить свой профиль (специалист)                              |
| GET   | `/clients/get-slots`         | Получить свободные слоты специалиста (клиент)                   |





## Технические особенности и безопасность

| Особенность | Описание |
|---|---|
| Каскадное удаление | При удалении пользователя автоматически удаляются все связанные данные (слоты, бронирования, платежи, отзывы, документы). Реализовано через `cascade="all, delete-orphan"` и `ondelete="CASCADE"`. |
| Пессимистическая блокировка | При бронировании слота используется `SELECT FOR UPDATE NOWAIT`, чтобы исключить одновременную запись от двух клиентов. Также добавлено уникальное ограничение на `slot_id` в таблице `appointment`. |
| Валидация загружаемых файлов | Проверка не только расширения, но и реального MIME-типа по сигнатуре (библиотека `filetype`). Запрещены исполняемые файлы и HTML‑документы. |
| Гибкая фильтрация и пагинация | Эндпоинт каталога специалистов поддерживает фильтры по специализации (поиск по подстроке, `ILIKE`), цене (`min_price`, `max_price`), рейтингу (`min_rating`), опыту (`min_experience`). Пагинация реализована через `page` и `per_page` (максимум 100). |
| Документация OpenAPI 3.0 | Все эндпоинты, модели данных и методы авторизации описаны в спецификации `openapi.yaml`, доступной через Swagger UI. |
| Безопасное хранение файлов | Загруженные файлы сохраняются в папку `uploads/` с уникальными именами (`avatar_{user_id}_{timestamp}.{ext}`), доступ к ним возможен только по API‑маршруту с проверкой прав. |
| Отказоустойчивость платежей | Вебхуки ЮKassa защищены HMAC‑подписью. При потере уведомления клиент может запросить актуальный статус через эндпоинт `GET /payments/check/<payment_id>`. |

## Интеграция с внешними сервисами

| Сервис | Назначение | Механизм |
|---|---|---|
| Logto | Аутентификация пользователей, выдача JWT-токенов. | Протокол OpenID Connect (authorization_code). Flask-сессия хранит `id_token`. |
| ЮKassa | Приём платежей, обработка возвратов, вебхуки. | REST API с идемпотентностью. Проверка подписи вебхуков `HMAC-SHA256`. Эндпоинт опроса (polling) для восстановления при сбоях. |
| Cal.com | Синхронизация расписания специалистов, создание бронирований. | API v2 (получение слотов, создание бронирований). Вебхуки для обновления статусов записей. |
| Jitsi | Организация видеоконференций. | Генерация уникальной ссылки `https://meet.jit.si/psych-help-{appointment_id}`, доступной только участникам за 10 минут до начала сессии. |


## Документация API (Swagger UI)
После запуска приложения интерактивная документация OpenAPI 3.0 доступна по адресу:

http://127.0.0.1:5000/docs

Чтобы авторизоваться и тестировать защищённые эндпоинты:

авторизируйтесь через /auth/login

Получите JWT-токен через POST /auth/get-token (тело {"email": "demo@example.com"}).

Нажмите кнопку Authorize и введите Bearer <токен>.

Выполняйте любые запросы прямо из Swagger UI.
